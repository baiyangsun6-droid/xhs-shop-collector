# 小红书商品采集器

本地可视化工具：输入小红书博主主页、店铺链接或 `xhslink.com` 短链，采集该博主当前店铺在售商品，并写入你已有的飞书多维表格。

默认采集模式是公开 HTTP 模式：不打开小红书页面、不自动点击、不滚动页面，也不使用你的小红书登录态。找不到公开店铺标识时会直接失败，避免继续用账号自动浏览。

## 环境要求

- macOS（桌面应用构建目前面向 Apple Silicon）
- Node.js 20 或更高版本
- 已安装并完成授权的 `lark-cli`，用于读写自己的飞书多维表格

## macOS 桌面应用

Apple Silicon Mac 可以直接构建桌面应用：

```bash
npm run package:mac
```

生成的 `release/mac-arm64/小红书商品采集器.app` 会完成本地临时签名，可以直接双击运行。应用会自动启动内嵌服务，不需要另开终端或手动打开网页；关闭应用后内嵌服务会一并退出。

桌面版配置保存在 `~/Library/Application Support/小红书商品采集器/data/config.json`。第一次从项目启动桌面版时，会自动迁移现有的 `data/config.json`。

## 启动

```bash
npm install
npm start
```

打开：

```text
http://127.0.0.1:3456
```

开发模式：

```bash
npm run dev
```

仓库只提供不含凭据的 `data/config.example.json`。启动后请在「连接配置」中填写自己的飞书信息，真实配置会保存在本机 `data/config.json`，该文件已被 Git 忽略。

## 飞书多维表格字段

默认字段名：

| 工具字段 | 飞书字段名 |
| --- | --- |
| 博主名字 | 博主名称 |
| 博主主页链接 | 博主主页链接 |
| 店铺链接 | 店铺链接 |
| 商品标题 | 商品标题 |
| 商品销量 | 商品销量 |
| 商品价格 | 商品价格 |
| 商品链接 | 商品链接 |
| 采集时间 | 采集时间 |

建议字段类型：

- 博主主页链接、店铺链接和商品链接可以使用 URL 或文本字段。
- 商品价格写入数字值；工具会把 `¥19.9` 写成 `19.9`。
- 商品销量使用数字字段；工具会把 `已售10`、`1.2万` 等文本写成纯数字，未展示销量时写 `0`。
- 采集时间使用日期时间字段。

## 使用步骤

1. 在「连接配置」里填写 `Base Token` 和 `Table ID / 表名`。
2. 选择飞书执行身份。推荐先用 `当前用户`，因为会按你自己的多维表格权限写入。
3. 在「数据映射」里确认字段名和飞书多维表格一致。
4. 点击「测试连接」确认字段映射正常。
5. 回到「任务中心」，粘贴一个或多个博主主页、店铺链接或短链。
6. 在「商品筛选」中选择全部商品，或选择按销量并填写阈值。
7. 点击「开始采集」。

「手动浏览」只用于你自己查看页面。默认采集不会使用这个浏览器，也不会复用登录态。

任务完成后，可以在每行的“更多”菜单中选择“从软件中删除”，也可以勾选多条任务后批量删除。该操作只清理本地任务、关联日志和商品预览，不会删除飞书多维表格中的数据；正在采集的任务需要先停止。

## 商品销量筛选

每次创建任务时都可以单独选择商品范围：

- 「全部商品」会写入店铺接口返回的全部在售商品。
- 「按销量」使用严格的大于条件。例如填写 `10`，只写入销量大于 `10` 的商品，销量等于 `10` 的商品不会写入。
- 如果筛选后没有符合条件的商品，工具会跳过飞书写入，不会清空该博主已有记录。

任务表中的商品数在启用筛选时显示为「符合数量/店铺总数」，任务详情会同时展示筛选条件和两个数量。

## 飞书权限

当前工具写入多维表格需要这些 scope：

```text
base:field:read
base:record:read
base:record:create
base:record:update
base:record:delete
```

本机已授权用户身份后，配置会保存为 `feishu.identity = "user"`。如需重新授权：

```bash
lark-cli auth login --scope "base:field:read base:record:read base:record:create base:record:update base:record:delete"
```

如果使用 `应用机器人` 身份，除了开放平台开通以上记录权限，还要确认该应用或机器人对目标多维表格有可编辑权限。否则会出现 `HTTP 403 / code 91403`。

## 覆盖逻辑

工具按规范化后的「博主主页链接」查找旧记录，先删除该博主上一次采集结果，再写入本次采集到的商品。

每条商品记录都会把本次实际采集的规范化店铺地址写入「店铺链接」字段；博主主页地址仍单独保存在「博主主页链接」。

同一个商品出现在不同博主店铺下时，会按「博主主页链接 + 商品链接」保留多条业务记录。

## 模拟模式

「连接配置」里可以开启模拟模式。模拟模式不会请求小红书，也不会写入飞书，只用于验证任务表、日志和商品预览流程。

## 浏览器兜底

「连接配置」里的浏览器兜底默认关闭。开启后，工具才会使用可见浏览器自动打开、点击和滚动页面来尝试找店铺入口。

注意：这类自动化浏览可能触发平台风控。收到账号违规预警后，不建议继续使用个人账号做自动化采集。

## 说明

- 工具不会绕过验证码或风控，也不会做反检测处理。
- 采集范围是店铺 Web/H5 接口当前返回的在售商品。默认只调用公开店铺分页接口；接口不可用或无法识别店铺时不会自动用账号继续尝试。
- 小红书页面结构可能变化，真实采集时如字段缺失，需要根据实际页面调选择器。
- 请遵守小红书、飞书及所在地适用的服务条款和法律，只处理自己有权访问和使用的数据。
- 不要提交 `data/config.json`、浏览器资料、调试截图或任何包含 Token、Cookie、个人信息的文件。
